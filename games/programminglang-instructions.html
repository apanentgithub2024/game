Syntax

Defining a variable

To define a variable, you can always use the define keyword. You're going to probably say "well, duh" at the next sentence. This keyword creates a variable, and assigns it a value. However, a value must always be defined for the variable. Here are the examples of the valid and invalid declarations.

Valid:
  • define variable_name = "Hello, world!"
  • define variable_name = 123
  • define define = "This still works! This might be confusing, or not recommended, though."

Notice the third valid example? This is not what many programming languages allow you to do. This is because the define keyword does not exist in all JavaScript engines. This is also transpiled into JavaScript, too. However as the string value mentions, it is not recommended to use that variable name, because if you use it, and if it is not used by a keyword, the compiler would become "confused".

Invalid:
  • define 1variable = "That does not work, because the variable cannot start with a number."
  • define variable
  • define variable$wow = "That also does not work."

As long as you understood the examples above, you can provide a valid and good variable declaration.

Locking a variable

If you're confused of what this actually means in the programming language, this just basically makes the variable constant (e.g. a constant variable cannot have it's value changed). This is also what not many programming languages allow you to do. This is done by using the lock keyword. If that variable doesn't exist, you may see an error that looks something like this:

CompilerError: The variable "invalid_var" has to be defined in order to be locked

However, this does not provide a source of where the error occurred in the compiler. If you don't see it, try pressing F12 or right-clicking then clicking "Inspect" or pressing/holding three keys: CTRL+Shift+I. Here are examples to do it:

Valid:
  • define variable_name = "Hello, world!"
    lock variable_name
  • define variable_name = 123
    lock variable_name
  • define define = "This still works! This might be confusing, or not recommended, though."
    lock define

Using define is valid, because it is used in/by a keyword.

Invalid:
  • define 1variable_name = "Hello, world!"
    lock 1variable_name
  • define variable_name
    lock variable_name
  • lock variable_name
